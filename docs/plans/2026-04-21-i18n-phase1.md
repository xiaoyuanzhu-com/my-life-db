# i18n Phase 1 Implementation Plan — Web Foundation

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Install `react-i18next`, extract all hardcoded English UI strings in the web frontend into namespaced JSON catalogs, wire `useTranslation` throughout, and teach the Go backend to inject a locale hint into `index.html`. After Phase 1, the app renders identically in English — but every string is routed through the i18n layer, ready for Phase 2 (zh-Hans translations).

**Architecture:** `react-i18next` configured in SPA-mode with:
- English fallback catalog bundled inline (`frontend/app/lib/i18n/fallback/en.json`) — used on cold boot and whenever a key is missing in the active locale.
- All other locales lazy-loaded from `/locales/{lang}/{namespace}.json` via `i18next-http-backend`.
- Active locale read from `<html lang>` attribute, which the Go backend injects based on `preferences.language` in SQLite (or `Accept-Language` header if unset).
- Go backend templates `index.html` on every HTML request and injects `<link rel="preload" as="fetch">` hints for the active locale's initial namespaces.

**Tech Stack:** React 19, TypeScript 5.9, Vite 6, react-i18next, i18next-http-backend, Go 1.25, Gin, `html/template`.

**Reference:** Design doc: `docs/plans/2026-04-21-i18n-design.md`.

---

## Design constraints (read before starting)

1. **No visible UI change.** After Phase 1, every page must render identical text to `main`. Verify via manual QA or screenshot diff.
2. **Single source of truth for active locale is the `<html lang>` attribute.** JS reads it once at boot; i18next uses it as the initial language. Do NOT hardcode `'en'` in the i18next config.
3. **English catalog is bundled inline** as a TypeScript import from `fallback/en.json`. It is also the fallback language (`fallbackLng: 'en'`). This guarantees the app renders before any network fetch resolves.
4. **All other languages are lazy-loaded**. `i18next-http-backend` fetches `/locales/{{lng}}/{{ns}}.json`. These JSONs live in `frontend/public/locales/` so Vite copies them into `dist/` at build time, and Go's static handler serves them as-is.
5. **Namespaces** split by feature so each page only needs the namespaces it uses. v1 namespaces: `common`, `settings`, `data`, `agent`, `explore`, `errors`. Phase 1 creates stub `errors.json` but does not migrate error handling (that's Phase 3).
6. **Key naming:** dot-separated, lower_snake or camelCase consistent within a namespace. Use `common.nav.data` not `common.navData` — nesting maps to natural JSON structure. **Always pass a default string** as the second arg: `t('nav.data', 'Data')`. This way if the key isn't defined, the English default still renders (belt + suspenders with the bundled fallback).
7. **No backend code changes** beyond the HTML templating in this phase. Error codes, `preferences.language` read/write API, and the language selector UI are Phase 2/3.

```ts
// frontend/app/lib/i18n/config.ts
import i18n from 'i18next';
import HttpBackend from 'i18next-http-backend';
import { initReactI18next } from 'react-i18next';
import { DEFAULT_UI_LOCALE, isSupportedLocale } from './metadata';
import fallbackEn from './fallback/en.json';

// Read active locale from the <html lang> attribute that the Go backend
// injects on every HTML response. If absent or unsupported, fall back to
// the default. This is the ONE place locale is sourced.
function readActiveLocale(): string {
  if (typeof document === 'undefined') return DEFAULT_UI_LOCALE;
  const lang = document.documentElement.lang?.trim();
  if (lang && isSupportedLocale(lang)) return lang;
  return DEFAULT_UI_LOCALE;
}

void i18n
  .use(HttpBackend)
  .use(initReactI18next)
  .init({
    lng: readActiveLocale(),
    fallbackLng: 'en',
    // Namespaces MUST be listed here so i18next knows which to load on boot.
    // Add more namespaces over time; not all need to be preloaded.
    ns: ['common'],
    defaultNS: 'common',
    supportedLngs: ['en', 'zh-Hans'],
    // Bundle English inline so the app renders instantly even if the network
    // fetch for the active locale is slow or fails.
    resources: {
      en: fallbackEn,
    },
    // Only non-English locales go through the HTTP backend.
    // partialBundledLanguages lets us mix inline + fetched.
    partialBundledLanguages: true,
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
    interpolation: {
      escapeValue: false, // React already escapes
    },
    react: {
      useSuspense: false, // don't wrap the whole app in Suspense for i18n
    },
    returnNull: false,
  });

export default i18n;
```

**Step 5.3: Implement `ResolveUILocale` in `backend/api/locale.go`**

```go
package api

import (
    "strings"

    "golang.org/x/text/language"
)

// ResolveUILocale picks the UI locale in priority order:
//   1. userPref if it's non-empty and in supported
//   2. best match from Accept-Language header among supported
//   3. fallback
//
// supported is the list of BCP 47 tags the frontend actually has catalogs for.
func ResolveUILocale(userPref, acceptLang string, supported []string, fallback string) string {
    if userPref != "" {
        for _, s := range supported {
            if strings.EqualFold(userPref, s) {
                return s
            }
        }
    }

    if acceptLang != "" {
        // Parse supported tags once.
        supportedTags := make([]language.Tag, 0, len(supported))
        for _, s := range supported {
            supportedTags = append(supportedTags, language.Make(s))
        }
        matcher := language.NewMatcher(supportedTags)

        userTags, _, err := language.ParseAcceptLanguage(acceptLang)
        if err == nil && len(userTags) > 0 {
            _, idx, confidence := matcher.Match(userTags...)
            if confidence >= language.High {
                return supported[idx]
            }
        }
    }

    return fallback
}
```

```go
// backend/api/spa.go
package api

import (
    "html/template"
    "net/http"
    "path/filepath"
    "sync"

    "github.com/gin-gonic/gin"
)

var (
    spaTplOnce sync.Once
    spaTpl     *template.Template
    spaTplErr  error
)

func loadSpaTpl(path string) (*template.Template, error) {
    spaTplOnce.Do(func() {
        spaTpl, spaTplErr = template.ParseFiles(path)
    })
    return spaTpl, spaTplErr
}

// ServeSPAIndex renders index.html with the correct <html lang> attr
// and locale-preload hints for react-i18next.
func (h *Handlers) ServeSPAIndex(c *gin.Context) {
    tpl, err := loadSpaTpl(filepath.Join("frontend", "dist", "index.html"))
    if err != nil {
        c.String(http.StatusInternalServerError, "template load error")
        return
    }

    // Read preferences.language from settings (empty string if unset).
    userPref, _ := h.server.DB().GetSettingString("preferences_language")

    lang := ResolveUILocale(
        userPref,
        c.GetHeader("Accept-Language"),
        []string{"en", "zh-Hans"},
        "en",
    )

    // Preload the `common` namespace so the nav renders without FOUC.
    // Additional namespaces are loaded lazily as routes are visited.
    preload := []string{}
    if lang != "en" {
        preload = append(preload, "/locales/"+lang+"/common.json")
    }

    c.Header("Content-Type", "text/html; charset=utf-8")
    c.Header("Cache-Control", "no-store") // HTML is per-user; don't cache
    _ = tpl.Execute(c.Writer, struct {
        Lang            string
        PreloadCatalogs []string
    }{lang, preload})
}
```
